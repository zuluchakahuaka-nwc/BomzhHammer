# AGENTS.md — БомжХаммер (BomzhHammer)

## 0. Правила работы

1. **Рефакторинг кода.** Регулярно проводи рефакторинг — выноси повторяющуюся логику в отдельные функции/классы, упрощай запутанный код, удаляй мёртвый код. Чистый код приоритетнее быстрого кода.
2. **Не раздувать файлы.** Не расширять бесконечно один и тот же файл. Если файл превышает ~400 строк — разбивай на модули, выноси логику в отдельные скрипты. Один файл = одна ответственность.
3. **Проверяй сам перед тем как говорить "работает".** Не отправляй пользователя проверять — запускай, читай логи, убеждайся сам. Пока сам не проверил — не утверждай что исправлено.
4. **Папка проекта:** `D:\Projects\BOMZHHAMMER`
5. **Справочная папка (списывай паттерны):** `D:\Projects\CWG`
6. **ТЕСТИРОВАНИЕ ОБЯЗАТЕЛЬНО.** После ЛЮБОГО изменения кода или данных — запускай тесты. Если тесты не прошли — исправляй молча. Только когда ВСЕ тесты зелёные — сообщай результат пользователю.
7. **Запуск тестов:** `& "D:\Projects\CWG\godot_bin\Godot_v4.2.2-stable_win64_console.exe" --headless --path "D:\Projects\BOMZHHAMMER" --script tests/test_runner.gd`
8. **Запуск игры (проверка):** `Start-Process "D:\Projects\CWG\godot_bin\Godot_v4.2.2-stable_win64.exe" -ArgumentList "--path","D:\Projects\BOMZHHAMMER"`
8a. **Запуск игры С БОТОМ:** `Start-Process "D:\Projects\CWG\godot_bin\Godot_v4.2.2-stable_win64_console.exe" -ArgumentList "--path","D:\Projects\BOMZHHAMMER","--bot"` — бот автоматически играет вместо игрока. Без `--bot` бот НЕ включается, игрок управляет сам.
9. **Названия карт ВСЕГДА переносятся на несколько строк.** Никогда не обрезать текст, не уменьшать шрифт. Использовать `_wrap_text()` для переноса длинных названий и описаний. Правило работает для ВСЕХ карточек — отряды, ситуации, заклинания, командиры, достижения.
10. **Тесты покрывают:** JSON-данные (целостность, уникальность ID, обязательные поля), изображения (все карты, иконки, фреймы, карта мира), connections (каждая территория связана), scenes (.tscn), scripts (.gd), ErrorParser (autoload).
11. **ErrorParser** — глобальный перехватчик ошибок. Подключён как autoload (второй после Logger). Каждый скрипт использует `ErrorParser.validate_node()` в `_ready()` для проверки дочерних узлов.
12. **БЕЛЫЙ ТЕКСТ НА КАРТАХ — АБСОЛЮТНОЕ ПРАВИЛО.** ВЕСЬ текст на ВСЕХ картах (название, описание, ATK/DEF/HP, лейблы) ВСЕГДА белый `Color(1, 1, 1)` с чёрной обводкой `Color(0, 0, 0)`. Никаких серых, тёмных, цветных текстов. Паттерн: `draw_string_outline(font, pos, text, align, width, size, outline_size, Color(0, 0, 0))` затем `draw_string(font, pos, text, align, width, size, Color(1, 1, 1))`. Работает для card_widget.gd и ЛЮБОГО кода рисующего текст на картах. НАРУШЕНИЕ = КРИТИЧЕСКИЙ БАГ. Это правило повторялось 10 раз — больше НЕ повторять.
13. **НЕ ЛОМАТЬ ПРЕДЫДУЩИЕ ИЗМЕНЕНИЯ.** После КАЖДОГО изменения:
    - Запусти `test_runner.gd` — все тесты должны пройти
    - Запусти игру через КОНСОЛЬ (`Godot_v4.2.2-stable_win64_console.exe`) на 5-10 секунд и проверь что НЕТ ошибок `SCRIPT ERROR` или `Parse Error`
    - Убедись что сцена загружается (видна карта, кнопки, ресурсы)
    - Только после этого сообщай результат
    - **ПРАВИЛО ОДНОГО ИЗМЕНЕНИЯ:** Делай одно изменение → проверяй → потом следующее. Не делай 5 изменений подряд без проверки между ними.
9. **Тест-бот игрового цикла:** `& "D:\Projects\CWG\godot_bin\Godot_v4.2.2-stable_win64_console.exe" --headless --path "D:\Projects\BOMZHHAMMER" --script tests/test_game_bot.gd` — бот автоматически тестирует: старт игры, создание колод, набор руки, розыгрыш карт, фазы, территории, полный игровой цикл (3+ хода). Использует `_process()` для доступа к autoloads (они недоступны в `_init()` и `_ready()` не вызывается в `--script` режиме). Если бот упал — читай логи, исправляй молча, перезапускай, пока не пройдёт.
10. **Принцип «запусти-исправь-перезапусти»:** После любого изменения запусти бота. Если упал — читай логи, исправь код/данные, перезапусти. Не сообщай результат пока бот не пройдёт чисто.
11. **Генерация изображений — через Pollinations AI API** (бесплатно, без ключа). Формат: PowerShell `Invoke-WebRequest` на `https://image.pollinations.ai/prompt/{encoded_prompt}?width=512&height=768&nologo=true`. **КАЖДЫЙ промпт должен быть ПОЛНОСТЬЮ уникальным** — разное описание, разные детали, разные предметы, разный ракурс. Pollinations кэширует одинаковые промпты и возвращает идентичные картинки. Если промпты похожи — результат будет одинаковый. Базовый стиль: `Dark satirical 2D illustration, post-soviet underground style, caricature, muted dirty colors, cardboard and trash aesthetic`. Формат файлов: JPG, 512x768. Папки: `assets/sprites/cards/units/`, `assets/sprites/cards/situations/`, `assets/sprites/cards/commanders/`, `assets/sprites/cards/spells/`, `assets/sprites/cards/inventions/`. Имя файла = ID карты. Генерировать по одной картинке, не массово (лимиты API). Не использовать процедурные заглушки (круги, квадраты, силуэты).
12. **Внизу экрана (рука) показываются ТОЛЬКО карты из руки игрока.** НЕ добавлять отряды с выбранной территории. Отряды на территориях видны через маркеры на карте (жёлтые/зелёные). Нарушение = критический баг с дублированием отрядов.
13. **Карта города НЕ должна пропадать.** Текстура `_city_map_tex` загружается через `load()` из `res://` путей. Если карта пропадает — это критический баг рендеринга.
14. **Религия спрашивается при 3+ территориях, идеология при 5+.** Проверка должна происходить КАЖДЫЙ ход в конце хода (фаза 5), а не только в фазе боя.
15. **Свои территории — жёлтая заливка, яркая и заметная.** Вражеские — красные. Нейтральные — коричневые. Цели атаки — жёлтая кайма/обод.
16. **Отображение отрядов при клике на территорию.** Игрок щёлкает ЛЕВОЙ кнопкой по СВОЕЙ территории → внизу экрана (панель руки) показываются карточки отрядов РАЗМЕЩЁННЫХ НА ЭТОЙ территории. Щёлкает другую территорию → внизу обновляются карточки отрядов с НОВОЙ территории. Один отряд = одна территория. Отряд не может быть везде и не может быть нигде. Если на территории нет отрядов — внизу пусто (только карты руки). Если отряды есть — их карточки видны внизу. Нарушение = критический баг.
17. **Запуск с таймаутом.** При запуске игры или бота через `Start-Process` — всегда закрывай предыдущий сеанс игры (`Get-Process Godot* | Stop-Process`). Если игра/бот зависли (нет логов 5+ минут) — убивай процесс, читай логи, перезапускай. Бот должен пропускать заставки нажатием пробела.

## 4. Идеологии (Политический строй)

Идеология выбирается при захвате 5-й территории (свалки) и определяет стиль управления империей бомжей. До выбора идеологии игрок играет без бонусов.

### 4.1 Алкоголизм

| Параметр | Значение |
|----------|----------|
| Тип | Тоталитарная плановая экономика |
| Налоги | 0 |
| Счастье | Контролируется водкой |
| **Бонус** | **+1 карта отряда** при доборе из правой колоды (вместо 2 берёт 3) |
| Минус | Риск "Алкогольной комы" — подданные не могут работать |
| Особенность | Можно варить самогон, строить самогонные реакторы |
| Армия | Алколесницы, спиртанки, снаряды-полторашки |

### 4.2 Дерьмократия

| Параметр | Значение |
|----------|----------|
| Тип | Демократия с высокими прибылями |
| Налоги | Средние |
| Счастье | Свободный рынок мусора |
| **Бонус** | **+25% к мелочишке** (прибыль) |
| Минус | Коррупция, риск "Жадность фраера сгубила" (-25 к торговле) |
| Особенность | Дипломатия + бонус, можно "сообразить на троих" |
| Армия | Торговые отряды, наёмники |

> **Важно:** Алкоголизм и Дерьмократия — это И идеология (постоянный эффект), И карта ситуаций (+25 к производству/прибыли).

## 5. Религии

Религия выбирается/открывается как достижение и даёт глобальные модификаторы.

### 5.1 Многобомжие

| Параметр | Значение |
|----------|----------|
| Тип | Политеизм (много богов-бомжей) |
| Описание | Верования в древних бомжей-покровителей. Каждый бог отвечает за свою сферу: бог свалки, бог вокзала, бог картона, бог полторашки |
| Бонус | +敬意 (уважение) в зависимости от бога-покровителя |
| Ритуалы | Подношения из просрочки на алтарь из картона |
| Особенность | Разные боги дают разные бонусы к территориям (бог свалки = +защита свалок, бог вокзала = +мелочишка) |

### 5.2 Трезвость

| Параметр | Значение |
|----------|----------|
| Тип | Аскетизм, отказ от алкоголя |
| Описание | Путь воздержания. Последователи не пьют, но работают эффективнее |
| Бонус | +к производительности, +к строительству, -к бунтам |
| Минус | Нельзя использовать карты "Опохмел", "Водка", "Самогонный реактор" |
| Ритуалы | Сбор бутылок ТОЛЬКО для сдачи, не для употребления |
| Особенность | "Самосознание +10" при "Бухнуть в одного" — трезвые бомжи более осознанны |

## 8. Система карт

### 8.1 Типы карт

| Тип | Описание | Колода |
|-----|----------|--------|
| **Отряд** (Unit) | Боевые единицы с рангами. ATK, DEF, HP | **Правая колода** — правая рука воюет |
| **Ситуация** (Situation) | События, достижения, эффекты | **Левая колода** — левая рука управляет |
| **Наука** (Science) | Открывает новые постройки/технологии | Левая колода (подтип ситуации) |
| **Заклинание** (Spell) | Накладываемые эффекты | Левая колода |
| **Командующий** (Commander) | Легендарные личности | Левая колода |
| **Достижение** (Achievement) | Открываемые бонусы | Открывается как ситуация из левой колоды |
| **Квест** (Quest) | Цепочки заданий | Левая колода |

### 8.1a Две колоды — две руки

- **Левая колода (Управление)** — карты ситуаций, заклинания, командующие, достижения, квесты. Левая рука управляет империей.
- **Правая колода (Война)** — карты отрядов. Правая рука воюет. Отряды отправляются на захват территорий.

### 8.1b Игровая карта

- Карта территорий города (свалки, вокзалы, притоны, помойки). Будет нарисована позже.
- Кликабельные территории. Захват — отправкой отрядов из правой колоды.

### 8.2 Формат данных карты отряда (JSON)

```json
{
  "id": "unit_bomzh_novice",
  "name": "Бомж-новичок",
  "type": "unit",
  "subtype": "infantry",
  "rank": "novice",
  "rarity": "common",
  "attack": 1,
  "defense": 1,
  "hp": 2,
  "cost": {
    "cans": 1,
    "vodka": 0,
    "coins": 0,
    "cardboard": 0
  },
  "description": "Только пришёл на свалку. Слаб, но бесплатен.",
  "flavor": "\"Где тут картон постелить?\"",
  "tags": ["infantry", "novice"],
  "linked_events": []
}
```

### 8.3 Формат заклинания (JSON)

```json
{
  "id": "spell_pohmele",
  "name": "Похмелье",
  "type": "spell",
  "target": "ally_or_enemy",
  "effect": "buff_or_debuff",
  "description": "Усиление войск: +50% атака на короткий период. Или наложение на противника: -сила",
  "cost": {
    "vodka": 2
  }
}
```

## 16. Ходовая система

### Фазы хода (по порядку):

1. **Фаза ресурсов** — доход от территорий (свалки дают банки/картон, вокзалы — мелочишку)
2. **Фаза розыгрыша** — взять 3-5 карт из колоды, разыграть
3. **Фаза перемещения** — двигать отряды по карте
4. **Фаза боя** — разрешить столкновения на спорных территориях
5. **Фаза событий** — случайные события (облавы, бродяги)
6. **Фаза конца хода** — проверить условия победы, обновить ресурсы

## 17. Боевая система

1. Атакующий и обороняющийся сравнивают отряды
2. Каждый отряд имеет ATK, DEF, HP
3. Базовый урон = ATK атакующего - DEF обороняющегося (min 0)
4. Модификаторы территории добавляются к DEF
5. Карты заклинаний могут модифицировать значения
6. Бросок d6 добавляется к ATK (случайность)
7. HP уменьшается, при HP <= 0 отряд уничтожен
8. Контратака если обороняющийся выжил

### 17.1 Наглость (naglost)

Некоторые отряды имеют параметр **наглость** (naglost) — это особая боевая характеристика.

**Принцип работы:**
- Каждый **второй** раз когда отряд атакует, наглость срабатывает как **дополнительная атака** — значение наглости приплюсовывается к урону.
- Но в **50% случаев** наглость **не срабатывает** (бросок монетки: орёл — работает, решка — нет).
- Итого: наглость добавляет урон каждый второй ход с 50% шансом = в среднем +25% от значения наглости к DPS.

**Пример:** Отряд с наглость 5 атакует. Ход 1 — обычная атака. Ход 2 — бросок 50/50: если успех, +5 к урону; если провал — обычная атака. Ход 3 — обычная. Ход 4 — снова бросок за наглость. И так далее.

## 21. UI-правила (обязательные)

### Управление картами в руке

| Действие | Что происходит |
|----------|----------------|
| **Левый клик** по карте отряда/командира | Карта прилипает к курсору (полупрозрачный фантом). Перемещай мышь над территорией и **отпусти левую кнопку** — отряд размещён. Отпустишь не над территорией — отмена. Escape тоже отменяет. |
| **Левый клик** по карте ситуации/заклинания | Карта мгновенно разыгрывается (эффект применяется) |
| **Правый клик** по ЛЮБОЙ карте | Увеличенная карта по центру экрана — можно прочитать описание, свойства, характеристики. Клик по фону или кнопка X — закрыть |

### Размещение отрядов на карте

1. Левый клик по отряду → фантом карты следует за курсором
2. Наведи на территорию (прямоугольник на карте) и **отпусти левую кнопку** → отряд размещён
3. Если на территории есть **NPC-гарнизон** (зелёный маркер `[xN]`) — бой разрешается мгновенно
4. Если победил — территория захвачена, маркер NPC исчезает
5. Если проиграл — твой отряд погиб, территория остаётся за NPC
6. Escape или отпускание кнопки не над территорией — отмена размещения

### Маркеры на территориях

| Маркер | Цвет | Значение |
|--------|------|----------|
| `xN` (жёлтый) | `Color(1, 1, 0.3)` | Твои отряды на территории |
| `[xN]` (зелёный) | `Color(0.2, 1.0, 0.3)` | NPC-гарнизон (противник) |

### NPC-гарнизоны

- Каждая нейтральная территория при старте содержит **1-5 случайных отрядов NPC**
- Каждая вражеская территория при старте содержит **3-10 случайных отрядов NPC**
- Карты отрядов противника отображаются с **зелёной толстой каймой** для отличия от своих
- Гарнизон виден при клике на территорию (панель инфо показывает количество)

### Z-порядок на экране карты

1. **Карта города** — нижний слой, границы чёткие
2. **Нижняя панель** — ресурсы и статус
3. **Карты в руке** — поверх всего
4. **Popup деталей карты** — модальное поверх всего (правый клик)
